## 4. 已落地内容

### 4.1 RVD 批量执行

新增 `RvdBatchStrategyExecutor`，把可检测字段转换成长表，将 30 个有向字段对配置广播到执行端，通过一次批量聚合边界生成各策略结果。`StrategyExecutionService` 自动识别 RVD 一对多计划并保持原计划顺序、策略标识、命中坐标和仓储写入语义。

集成测试逐项比较批量路径和原单策略路径的坐标、值哈希、原因、明细和分数，结果完全一致。最终 `flights` 仍为 30 个 RVD 计划，总策略命中 86874 条。

### 4.2 公共特征产物与复用

新增 `RahaFeaturePreparationService`、请求对象和结果对象，统一执行策略计划、策略命中和特征组装。SAMPLE 先准备一次特征，TRAIN 通过请求引用复用同一产物，不再重复执行完整策略链路。

当传播标签退化为不可训练、正例占比低于 0.05 或高于 0.95 时，训练服务回退到直接标签，避免字段模型因传播偏置静默缺失。本轮 4 个时间字段均生成并发布模型。

### 4.3 逐轮主动采样

新增 `ActiveSamplingOrchestrator`，按单轮预算 1 执行 20 轮。每轮先产生标注任务，再只读取该任务行的真值，保存直接标签并完成任务，随后使用更新后的覆盖状态进入下一轮。

`flights` 最终采样行顺序为：

```text
[1019, 1297, 1270, 1719, 172, 1222, 69, 13, 130, 1223,
 1724, 694, 1224, 696, 2082, 1225, 2084, 2085, 102, 1728]
```

共 20 行、120 个直接标签。未被选中的真值不进入策略、聚类、传播和训练输入。

小数据回归发现总预算可能超过候选行数，现已按稳定特征坐标自动收缩预算，并在容器验收中至少保留一行用于评测。

### 4.4 可扩展聚类

新增 `ScalableColumnClusterer`。样本数不超过精确上限时沿用层次聚类；超过上限时切换为确定性余弦分区聚类，稳定排序输入、稳定初始化中心、固定打破平局方式，并最多迭代 100 次。

该路径解决了 2376 行数据使用精确层次聚类长期不结束的问题。100 次上限保留了效果稳定性，但仍是当前主要性能成本之一。

### 4.5 三段隔离和字段阈值

新增 `EvaluationSplitService` 和 `EvaluationSplit`，先排除 120 个直接训练坐标，再使用稳定哈希将剩余坐标划分为阈值验证集和最终测试集。

最终划分如下：

| 分区 | 单元格数 |
| --- | ---: |
| 直接训练标签 | 120 |
| 阈值验证集 | 2877 |
| 最终测试集 | 11259 |

三者无坐标重叠。极小数据集哈希结果若全部落入同一侧，会按稳定单元格顺序移动一个坐标，保证验证集和测试集非空。

新增 `ThresholdSelectionPolicy`，对 `act_dep_time` 使用召回约束下的精确率优先排序。候选阈值为 0.01 至 0.99，其他字段保留 0.5。


### 4.6 逐策略坐标金标准

新增 Java 策略产物写出器、Python 策略缓存导出脚本和双边比较脚本。坐标统一转换为 `rowId|columnName`，排序后计算 SHA-256、交集、独有坐标、精确率、召回率和杰卡德系数。

该能力不再用“策略总数相同”代替语义一致，而是给每个可映射策略提供坐标级证据。

### 4.7 独立提交端和工作端

容器验证入口支持三种模式：

| 模式 | 行为 |
| --- | --- |
| `COMBINED` | 兼容模式，在同一应用内提交并消费 |
| `SUBMITTER` | 只注册和调用三个 UDF，轮询文件终态 |
| `WORKER` | 独立扫描共享队列并执行 SAMPLE、TRAIN、DETECT |

最终独立验证同时启动两个 Spark 应用，各限制 1 个执行核心。三个请求均生成 `.completed.request` 和 `.succeeded`，提交端没有直接调用采样、训练或检测服务。

## 10. `act_dep_time` 阈值验收

验证集选择结果：阈值 0.17、精确率 0.844203、召回率 0.723602、F1 0.779264。阈值 0.5 在同一验证集上的召回率为 0.711180，因此策略按规则使用 0.72 作为召回下限，并降低阈值满足约束。

| 阈值 | 真阳性 | 假阳性 | 假阴性 | 精确率 | 召回率 | F1 |
| ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| 0.50 | 1121 | 261 | 437 | 0.811143 | 0.719512 | 0.762585 |
| 0.17 | 1131 | 274 | 427 | 0.804982 | 0.725931 | 0.763415 |

阈值接入和独立选择机制本身正确，但业务目标没有完全达到：假阳性没有降低 20%，反而增加 13 个；F1 仅提高 0.000830。根因是当前模型在阈值 0.5 时已经低于 0.72 召回下限，现有策略优先补足召回，而不是降低误报。

后续不能继续只调阈值，应优先改善 `act_dep_time` 的特征、采样标签分布或概率校准。

4. 对固定快照持久化公共特征和聚类产物，跨任务和跨进程按版本加载。

## 17. 后续调整优先级

### 17.1 第一优先级：持久化任务上下文

把 SAMPLE 输出的特征版本、聚类版本、采样任务和直接标签引用写入共享仓储；TRAIN 只按引用加载。把候选模型、发布版本和阈值写入跨进程模型仓储；DETECT 只按发布模型版本加载。完成后才能执行工作端重启、双实例认领和幂等恢复测试。

### 17.2 第二优先级：改善 `act_dep_time` 模型

当前阈值已无法同时满足召回下限和误报下降。应增加概率校准、困难负样本采样和字段专用特征，并在多个稳定种子上比较。验收时必须在同一冻结测试集同时输出阈值 0.5 和候选阈值，不再只看验证集最优值。

### 17.3 第三优先级：压缩到 100 秒以内

先通过事件日志确定阶段占比，再优化聚类收敛和四字段训练作业。性能门禁应使用相同资源、相同快照的热态中位数，并保留冷态结果作为部署容量参考。

### 17.4 第四优先级：按金标准扩展策略

先修正 19 个近似 RVD 的语义差异，再实现带字符上限和控制字符过滤的动态 PVD。OD 只有在业务要求严格复刻 Python 时才投入 dBoost 等价实现，否则保持 Spark 原生近似并以效果门禁验收。

## 18. 回滚说明

1. 独立模式可通过 `fmdb.validation.mode=COMBINED` 回到兼容验证路径。
2. `act_dep_time` 阈值与模型版本绑定，可回滚到阈值 0.5 的上一发布版本。
3. TRAIN 请求未提供公共特征产物时仍可按原流程准备特征。
4. 可扩展聚类对小数据自动使用原精确层次聚类路径。
5. 策略扩展尚未设为默认，本轮仍保持 Java 生产口径 60 个策略。

生产环境不得以回滚到组合模式替代独立消费者持久化建设。
