### 4.3 依赖控制

Maven 构建已经增加以下控制：

- 必须使用 JDK 8 构建。
- Maven 版本不能低于 3.8。
- 禁止 POM 中出现同一依赖多个版本。
- 禁止引入 Spark 4。
- 禁止引入 `_2.13` Spark 组件。
- 使用 Java 8 API 签名检查，避免调用高版本 JDK API。
- MLlib 只能通过显式 `with-mllib` 配置启用。

默认 Spark 依赖树只包含 Spark 3.3.1 `_2.12` 组件。

## 5. 配置模块

配置模块位于：

```text
src/main/java/com/fiberhome/ml/raha/config/
```

当前配置对象包括：

- `RahaJobConfig`。
- `StrategyConfig`。
- `FeatureConfig`。
- `ModelConfig`。
- `ResourceConfig`。
- `FailureToleranceConfig`。
- `ConfigErrorCode`。
- `ConfigValidationException`。
- `RahaConfigValidator`。
- `ConfigVersioner`。

配置校验覆盖：

- 任务类型、数据集、输入引用和行标识必填检查。
- 结果保留天数检查。
- 策略族、策略数量、RVD 列对和策略超时检查。
- 字段白名单和黑名单冲突检查。
- 最大特征数量检查。
- 模型类型和阈值检查。
- 并发、广播、缓存和阶段超时检查。
- 策略失败比例和重试次数检查。

配置版本对集合进行排序后再生成摘要，因此字段集合和策略族的插入顺序不会导致版本漂移。

## 6. 领域对象模块

### 6.1 数据对象

| 对象 | 用途 |
| --- | --- |
| `RahaDataset` | 统一数据集、快照、模式、字段和列画像 |
| `ColumnMetadata` | 描述字段类型、序号、可检测和敏感属性 |
| `ColumnProfile` | 保存策略生成所需的列级统计 |
| `CellCoordinate` | 使用数据集、快照、行和列稳定定位单元格 |
| `CellValue` | 保存原始值、值哈希和脱敏值 |
| `DetectionResult` | 保存单元格检测判断、分数、策略和模型版本 |

### 6.2 策略和特征对象

| 对象 | 用途 |
| --- | --- |
| `StrategyPlan` | 保存可独立执行和重放的策略配置 |
| `StrategyHit` | 保存策略对单元格产生的候选错误信号 |
| `FeatureDefinition` | 保存稳定特征编号、名称、类型和来源 |
| `FeatureDictionary` | 冻结单列特征名称到编号的映射 |
| `SparseFeatureRow` | 保存单元格非默认特征和值 |

### 6.3 聚类、标签和模型对象

| 对象 | 用途 |
| --- | --- |
| `ClusterAssignment` | 保存单元格到列内聚类的映射 |
| `CellLabel` | 保存标签值、来源、置信度和传播关系 |
| `RahaColumnModel` | 保存列级模型及其模式、策略和特征版本 |

### 6.4 任务和阶段对象

| 对象 | 用途 |
| --- | --- |
| `RahaJob` | 保存任务生命周期、当前阶段和失败上下文 |
| `RahaStage` | 保存阶段类型、尝试序号、状态和失败上下文 |
| `JobStatus` | 限制任务状态转换 |
| `StageStatus` | 限制阶段状态转换 |
| `JobType` | 定义评测、训练、采样、检测和策略分析模式 |
| `StageType` | 定义 Raha 流程阶段 |

任务和阶段状态修改使用先校验后切换的顺序。参数或时间校验失败时，状态保持不变，避免出现部分更新。

## 8. MLlib 结论

Spark MLlib 使用 `pom.xml` 中与 Spark 3.3.1、Scala 2.12 一致的 `provided` 依赖。工程不检查本地 Jar 目录或运行时 Classpath；运行平台负责提供兼容组件，训练失败时由后续模型配置决定是否使用规则加权降级。
