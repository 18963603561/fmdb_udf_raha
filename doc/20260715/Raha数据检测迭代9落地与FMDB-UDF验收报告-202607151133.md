## 2. 实施结论

迭代 9 已建立不依赖旧工程源码和私有 Java 对象的 FMDB 适配层。数据读取和表写入只依赖 Spark 3.3.1 标准 `SparkSession`、Catalog 和 `Dataset<Row>` 接口，生产环境由 FMDB 的 SessionCatalog 和存储实现接管，核心 Raha 算法层不直接引用 FMDB 厂商类。

FMDB 适配已支持表和只读 SQL 两种输入，统一执行行标识非空唯一校验、模式哈希、字段范围解析和快照生成。任务状态和检测结果使用固定表模式及业务主键幂等追加。模型参数与特征字典按不可变版本保存和加载，同版本内容冲突会被拒绝。

三个 UDF 均为表级异步任务入口。UDF 只解析参数、检查幂等键、创建 `CREATED` 任务并返回 JSON，不在单元格调用中同步执行 Raha 全表算法。重复请求返回原 `jobId`，幂等键与配置冲突、非法参数和提交异常均返回稳定错误码并记录日志。

## 5. FMDB 数据适配

### 5.1 数据加载

`FmdbDatasetLoader` 继续实现核心 `RahaDatasetLoader` 接口。调用方通过 `DataFormat.FMDB_TABLE` 或 `DataFormat.FMDB_SQL` 指定来源：

- 表来源调用 `SparkSession.table`。
- SQL 来源只接受以 `SELECT` 或 `WITH` 开始的只读查询。
- 读取后校验行标识字段存在、非空且唯一。
- 根据 Spark 模式生成字段顺序、类型、可空、检测和敏感标记。
- 生成稳定模式哈希及数据快照。
- Catalog、SQL 或数据校验异常统一转换为稳定数据加载错误并记录上下文和堆栈。

### 5.2 表网关

`FmdbTableGateway` 抽象表存在判断、读取和幂等追加。生产实现 `SparkSqlFmdbTableGateway` 使用 Catalog 与 `saveAsTable`；开发测试实现 `InMemoryFmdbTableGateway` 使用 Spark 临时视图，避免 Windows 本机缺少 `winutils.exe` 时产生文件权限依赖。

两种实现都校验表名、数据模式和业务主键。幂等追加先对已有主键执行空值安全反连接，只写尚不存在的记录。

### 5.3 结果表

任务表保存任务标识、幂等键、任务类型、数据集、快照、配置版本、状态、阶段、时间及失败信息。任务审计主键为：

```text
job_id + status + current_stage_id
```

检测结果表保存单元格坐标、值哈希、可选脱敏值、错误判断、分数、阈值、策略、原因和模型版本。检测结果主键为：

```text
job_id + cell_id + model_version
```

结果表没有 `correct_value`、修复候选或清洗回写字段。

## 6. 模型与字典存储

`FmdbModelStore` 实现现有 `ColumnModelStore`，模型路径格式为：

```text
fmdb://<model_table>/<model_version>
```

模型表保存模型名称、版本、字段、分类器、字典版本、特征维度、阈值、截距、稀疏系数和训练模式。字典表按字典版本和特征编号保存字段、特征名称、类型、来源及默认值。

同一模型或字典版本再次保存时先加载现有内容：完全一致则幂等返回原路径，内容不同则拒绝覆盖，防止版本污染。加载结果按路径或版本缓存在当前进程，检测服务重启后仍可从 FMDB 表重建模型和字典。

## 7. UDF 接口契约

### 7.1 调用语义

三个 UDF 输入一个标准表单编码字符串，返回一个 JSON 字符串。函数由 `RahaUdfRegistrar` 一次性注册：

| 函数 | 语义 |
| --- | --- |
| `F_DW_RAHATRAIN` | 异步提交训练任务 |
| `F_DW_RAHADETECT` | 异步提交整表检测任务 |
| `F_DW_RAHASAMPLE` | 异步提交采样任务 |

调用方不得对输入表每一行调用这些函数。推荐使用单行 SQL 调用并将编码请求作为常量参数。

### 7.2 公共请求参数

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| `datasetId` | 是 | 逻辑数据集标识 |
| `inputReference` | 是 | FMDB 表名或 URL 编码后的只读 SQL |
| `sourceType` | 是 | `TABLE` 或 `SQL` |
| `rowIdColumn` | 是 | 稳定唯一行标识字段 |
| `snapshotId` | 否 | 指定输入快照 |
| `idempotencyKey` | 是 | 调用方稳定幂等键 |
| `caller` | 是 | 调用人或调用系统 |
| `resultTable` | 是 | 任务产物或结果表 |

任务专属参数：

| UDF | 参数 | 约束 |
| --- | --- | --- |
| 训练 | `annotationReference` | 必填标注表 |
| 检测 | `modelVersion` | 必填已发布模型版本 |
| 采样 | `labelingBudget` | 必须为正整数 |

未知参数、重复键、非法 URL 转义、非法表名、超过 65536 字符、缺失专属参数和跨任务参数都会被拒绝。

### 7.3 返回 JSON

返回字段固定为：

```json
{
  "jobId": "job-...",
  "taskType": "DETECT",
  "status": "ACCEPTED",
  "resultLocation": "fmdb://result_table/job-...",
  "configVersion": "...",
  "errorCode": null,
  "errorMessage": null,
  "submittedAt": 1000
}
```

状态语义：

| 状态 | 说明 |
| --- | --- |
| `ACCEPTED` | 新任务已保存为 `CREATED` |
| `DUPLICATE` | 相同幂等键和配置已存在，返回原 `jobId` |
| `REJECTED` | 参数、幂等冲突、运行时或提交失败，返回稳定错误码 |

UDF 返回结果不包含原始数据值、修复值或清洗后数据。

当前自动化测试使用 Spark 3.3.1 本地模式和 `InMemoryFmdbTableGateway` 模拟 FMDB 表语义，因为 Windows 测试机没有配置 Hadoop `winutils.exe`，无法对本地文件仓库执行 `saveAsTable` 权限操作。生产实现 `SparkSqlFmdbTableGateway` 已保留，并使用相同表模式和幂等主键调用平台 Catalog。

进入真实 FMDB 环境部署时必须执行以下检查：

1. 确认平台 Spark 3.3.1 与 Scala 2.12 运行环境可用。
2. 确认 Spark MLlib 3.3.1 `_2.12` 由平台提供。
3. 使用生产数据库和结果表执行一轮表读取、任务写入、模型写入及检测结果写入冒烟测试。
4. 在执行 UDF 的每个 JVM 初始化 `RahaUdfRuntime`，或注入可序列化的远程任务提交器。

权限校验、调用审计持久化、保留清理和生产性能验收属于迭代 10 的 `T125` 至 `T134`，不在本次任务范围内。
