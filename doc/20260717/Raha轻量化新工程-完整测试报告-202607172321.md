# Raha 轻量化新工程完整测试报告

## 1. 报告结论

Raha 轻量化新工程已完成 P0 至 P11 全部阶段实现，并在 Docker Compose 项目 `fmdb_udf_schmatch` 的 Spark 集群中完成采样、全量训练、增量训练、检测和评估实测。

最终验收数据为 `datasets/toy/dirty.csv` 和 `datasets/toy/clean.csv`。本次检测共评估 `Kingdom` 字段的 7 个单元，识别出 5 个真实错误，结果为真阳性 5、假阳性 0、假阴性 0，精确率、召回率和 F1 均为 1.0。

最终结论：构建通过、单元测试通过、Jar 部署通过、三类 UDF 入口调用通过、ORC 持久化通过、全量训练通过、增量训练通过、检测评估通过，可以作为 Raha 新工程基线继续集成。

### 7.2 采样验证

第一次调用 `F_DW_RAHASAMPLE` 未传 `targetColumns`，系统自动解析为全部业务字段 `ID`、`Lord`、`Kingdom`，选取 7 条待标注记录。

第二次调用显式指定 `targetColumns=Kingdom`，选取 7 条增量标注记录。结果证明请求既支持默认全字段，也支持指定字段。

行标识未要求用户提供必须存在的主键。测试数据使用 `ID` 作为可用行键；框架同时保留组合键和整行哈希降级模式，用于无主键或重复记录场景。

### 7.3 全量训练验证

全量训练未显式指定 `targetColumns`，系统按采样范围处理 `ID`、`Lord`、`Kingdom`。由于 `ID` 和 `Lord` 的 7 个样本均为同一标签类别，不满足逻辑回归双类别训练条件，系统记录可恢复告警并跳过这两个字段。

`Kingdom` 字段具有正常和错误两类标签，成功训练 1 个 `LOGISTIC_REGRESSION` 列模型，使用训练样本 7 条，并持久化模型集合、列模型和训练样本。

### 7.4 增量训练验证

增量训练使用两个采样批次的数据契约，当前批次显式训练 `Kingdom`，父模型为全量模型集合。系统完成父模型兼容性检查、样本合并、特征字典复用和新模型版本发布。

增量训练成功生成 1 个 `Kingdom` 列模型，训练样本 7 条。原全量模型集合保持不变，新模型集合通过父版本字段建立继承关系。

### 7.5 检测验证

检测请求未显式传入 `targetColumns`，系统默认使用增量模型集合中实际可用的全部模型字段，因此检测范围为 `Kingdom`。

## 9. 实测发现与修复

| 问题 | 原因 | 修复结果 |
| --- | --- | --- |
| Spark Java API 编译不兼容 | 使用了当前版本不支持的字符串数组重载和列取反方法 | 改为显式 `Column` 数组、连接条件和布尔比较 |
| 容器读取绝对 CSV 路径失败 | Spark 将无协议绝对路径按 HDFS 路径处理 | 本地绝对路径统一补充 `file:` 协议 |
| 报告写入挂载目录失败 | 容器用户对宿主机挂载目录无写权限 | 默认报告位置调整为容器 `/tmp` |
| 验证脚本不能直接复跑 | 默认数据路径和驱动网络参数与目标容器不一致 | 脚本改为实际挂载路径并补充驱动地址和 Hive 参数 |
| 训练结果字段范围不准确 | 结果曾包含因单类别被跳过的字段 | 结果仅记录实际成功训练的模型字段 |
| 动态分区配置产生重复告警 | 通过 Spark SQL 动态执行 Hive `SET` 不可靠 | 改为 Hadoop 配置注入，并在提交参数中显式设置 |
| 绝对路径评估读取失败 | 评估阶段未统一处理容器本地文件协议 | 评估读取同样补充 `file:` 协议 |

## 10. 剩余环境提示

Hive Metastore 在部分分区查询期间出现过一次 `TTransportException: Cannot write to null outputStream`，Spark 自动重连后继续执行，最终报告、表记录和 ORC 文件均完整生成。该问题属于现有容器中 Hive Metastore 连接稳定性提示，不是 Raha 算法或数据契约失败。

建议后续环境运维检查 Hive Metastore 进程资源、连接空闲关闭策略和网络稳定性。当前问题不阻塞本次功能验收，但在更大数据量或更高并发下应纳入稳定性观察。

最终回归的业务报告在 107060 毫秒时写入完成。外层 PowerShell 日志收集命令因 120 秒工具窗口结束而返回超时，但容器内 Spark 进程随后正常结束，报告文件时间、八表增量记录和检测结果均已复核。此前相同业务代码的完整容器回归已取得退出码 0；最后一次修改仅将动态分区参数从无效 SQL `SET` 移至 Hadoop 配置。
