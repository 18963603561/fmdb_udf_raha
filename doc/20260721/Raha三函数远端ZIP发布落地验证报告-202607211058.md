## 三、验证方式

原始 SQL 脚本位置：

```text
doc/20260720/person-info-raha-run-202607201851/sql
```

`spark-sql` CLI 直接执行 GenericUDF 时，会在 executor 线程初始化 UDF，而函数内部还需要驱动侧 SparkSession 再发起 Spark 作业，因此直接执行 SQL 脚本会失败。项目已有 `RahaUdfDriverApp` 专门用于这种驱动侧执行场景，本次使用 SQL 脚本同目录的 request 参数执行三函数。

为避免历史幂等任务复用，本次验证请求只调整了 `requestId`；检测请求额外替换为本次训练生成的 `modelSetVersion`。
