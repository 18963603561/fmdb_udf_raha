# Raha 三函数 ZIP 远端 Web 发布分析与改造方案

生成时间：2026-07-21 09:33

## 一、结论

当前 Raha 三个 UDF 函数已经具备 ZIP 生成能力，也会在 `publishZip=true` 时把 ZIP 名称和 ZIP 地址字段写入返回结果。

但是，当前实现默认没有通过 SSH 或 SCP 把 ZIP 发布到另一台 `fmdbManagerPod` 的 Web 服务目录，因此默认返回的不是可直接通过浏览器下载的 `http` 地址。

当前返回值的含义如下：

| 场景 | 返回结果 | 是否等同远端 Web 下载地址 |
| --- | --- | --- |
| `raha.output.web-base-url` 为空 | 返回 `localWebRoot` 下的本地绝对路径 | 否 |
| 请求或配置传入 `webBaseUrl` | 返回按 `webBaseUrl/yyyy/M/d/file.zip` 拼接的字符串 | 只在该目录已被对应 Web 服务真实映射时才是 |
| HDFS 上传成功 | 只把 ZIP 复制到 HDFS 导出目录 | 否 |
| 参考 schmatch 的远端发布 | SSH 建目录、SCP 上传、返回 `http://<fmdbManagerPod>:10030/fmdb_report/...zip` | 是 |

因此，如果目标是“ZIP 包通过 SSH 到另一台 `fmdbManagerPod`，再提供 URL 下载”，当前 Raha 代码还没有完成，需要补齐远端 Web 发布逻辑。

## 六、推荐改造方案

### 1. 保持三函数服务层不变

不修改以下入口和服务方法：

| 文件或方法 | 原因 |
| --- | --- |
| `F_DW_DETCOLLECT` | 已正确调用 `service.collect(...)` |
| `F_DW_DETTRAIN` | 已正确调用 `service.train(...)` |
| `F_DW_DETRUN` | 已正确调用 `service.detect(...)` |
| `RahaDetectionUdfService.collect` | 已传入 `spark.sqlContext()` 和发布配置 |
| `RahaDetectionUdfService.train` | 已在训练成功后发布训练报告 ZIP |
| `RahaDetectionUdfService.detect` | 已在检测成功后发布检测明细 ZIP |

### 2. 修改 `RahaZipWebPublisher.publish(...)`

当前代码逻辑是：

```java
zip(zipPath, files);
uploadHdfs(sqlContext, zipPath, config);
String url = publishLocal(zipPath, config);
return url;
```

建议改为：

```java
zip(zipPath, files);
uploadHdfs(sqlContext, zipPath, config);
String url = publishWeb(sqlContext, zipPath, config);
return url;
```

这里的 `publishWeb(...)` 先尝试远端 Web 发布，失败或不适合远端发布时再降级到本地发布。

### 3. 新增 `publishWeb(...)`

建议新增私有方法：

```java
private String publishWeb(SQLContext sqlContext,
                          Path zipPath,
                          RahaPublishConfig config) throws IOException {
    try {
        String remoteUrl = tryRemoteWeb(sqlContext, zipPath);
        if (!isBlank(remoteUrl)) {
            return remoteUrl;
        }
    } catch (Exception exception) {
        // 异常处理：远端 Web 发布依赖集群 SSH/SCP，失败后降级到本地 Web 目录。
        LOGGER.warn("Raha UDF 远端 Web 发布失败，降级到本地 Web 目录，zipPath={}",
                zipPath.toAbsolutePath(), exception);
    }
    return publishLocal(zipPath, config);
}
```

### 4. 新增 `tryRemoteWeb(...)`

完全参考 schmatch 的 SSH 和 Web 规则，建议新增：

```java
private String tryRemoteWeb(SQLContext sqlContext,
                            Path zipPath) throws Exception {
    if (sqlContext == null) {
        return "";
    }
    String hostConf = sqlContext.getConf("spark.driver.host");
    if (isBlank(hostConf) || isLocalHost(hostConf)) {
        return "";
    }
    LocalDate today = LocalDate.now();
    String fmdbManagerPod = hostConf.replaceFirst(
            "sql-(analysis|query)-([0-9]*)", "fmdbmanager-master-0");
    if (!isSafeRemoteHost(fmdbManagerPod)) {
        LOGGER.warn("Raha UDF 远端 Web 主机名包含非法字符，跳过远端发布，host={}",
                fmdbManagerPod);
        return "";
    }
    String webPath = "/opt/software/fmdb/manager/web/jetty/webapps/fmdb_report/"
            + today.getYear() + "/" + today.getMonthValue() + "/"
            + today.getDayOfMonth();
    runCommand("ssh", "-p", "22034", fmdbManagerPod, "mkdir", "-p", webPath);
    runCommand("scp", "-P", "22034", zipPath.toAbsolutePath().toString(),
            fmdbManagerPod + ":" + webPath);
    return "http://" + fmdbManagerPod + ":10030/fmdb_report/"
            + today.getYear() + "/" + today.getMonthValue() + "/"
            + today.getDayOfMonth() + "/" + zipPath.getFileName();
}
```

说明：

1. `spark.driver.host` 为空时，不做远端发布。
2. `spark.driver.host` 是本机地址时，不做远端发布。
3. `sql-analysis-*` 和 `sql-query-*` 会被替换为 `fmdbmanager-master-0`。
4. 如果主机名包含非法字符，则跳过远端发布。
5. SSH 和 SCP 端口固定为 `22034`。
6. Web 下载端口固定为 `10030`。
7. Web 路径固定使用 `/fmdb_report/yyyy/M/d/file.zip`。

### 5. 新增外部命令执行方法

建议新增：

```java
private void runCommand(String... command) throws IOException, InterruptedException {
    Process process = new ProcessBuilder(command).redirectErrorStream(true).start();
    try (InputStream inputStream = process.getInputStream()) {
        while (inputStream.read() >= 0) {
            // 外部命令输出不进入返回结果，仅用于避免缓冲区阻塞。
        }
    }
    int exitCode = process.waitFor();
    if (exitCode != 0) {
        throw new IOException("命令执行失败 exitCode=" + exitCode
                + " command=" + Arrays.toString(command));
    }
}
```

### 6. 新增主机校验方法

建议新增：

```java
private boolean isSafeRemoteHost(String host) {
    return !isBlank(host) && host.matches("[A-Za-z0-9._-]+");
}

private boolean isLocalHost(String host) {
    if (isBlank(host)) {
        return true;
    }
    String normalized = host.trim().toLowerCase(Locale.ROOT);
    if ("localhost".equals(normalized)
            || "127.0.0.1".equals(normalized)
            || "0.0.0.0".equals(normalized)) {
        return true;
    }
    try {
        String localHostName = InetAddress.getLocalHost().getHostName();
        return normalized.equalsIgnoreCase(localHostName);
    } catch (Exception exception) {
        return false;
    }
}
```

## 七、关于 `webBaseUrl` 的处理建议

当前 Raha 已经存在 `webBaseUrl` 配置，参考 schmatch 没有这个配置。

推荐处理方式：

1. 远端发布成功时，优先返回参考实现生成的 `http://<fmdbManagerPod>:10030/fmdb_report/...zip`。
2. 远端发布跳过或失败时，继续使用当前 `publishLocal(...)`。
3. 当前 `publishLocal(...)` 内的 `webBaseUrl` 拼接逻辑可以保留，作为本地降级发布的兼容能力。

这样可以做到：

| 目标 | 结果 |
| --- | --- |
| 满足本次远端发布诉求 | 可以 |
| 保持现有请求参数兼容 | 可以 |
| 不影响三函数服务层 | 可以 |
| 不破坏已有 README 契约 | 可以 |

如果要求与 schmatch 逐字一致，也可以删除 `webBaseUrl` 拼接，让本地降级永远返回本地绝对路径。但这会改变当前 Raha 已公开的配置语义，不建议作为第一版改造。

## 八、建议改造后的发布流程

```text
三函数调用 zipPublisher.publish
生成本地 ZIP
尝试上传 HDFS
读取 spark.driver.host
判断是否适合远端发布
推导 fmdbmanager-master-0
通过 SSH 创建远端日期目录
通过 SCP 上传 ZIP
返回远端 HTTP 下载 URL
远端失败时复制到本地 Web 根目录
返回本地路径或兼容的 webBaseUrl URL
```

## 十、风险点

| 风险 | 说明 | 建议 |
| --- | --- | --- |
| SSH 或 SCP 命令不存在 | 运行容器内必须有 `ssh` 和 `scp` | 在镜像或运行环境中确认命令可用 |
| 免密访问不可用 | `ssh -p 22034 <pod>` 可能要求密码 | 沿用 schmatch 的集群信任配置 |
| `spark.driver.host` 不是 `sql-analysis-*` 或 `sql-query-*` | 替换后可能仍是原主机 | 与参考实现保持一致，先不额外扩展 |
| 远端目录无写权限 | SCP 会失败 | 失败时自动降级本地发布，并记录 warn 日志 |
| Web 服务未映射目录 | 文件已上传但 URL 不能下载 | 确认 Jetty 的 `fmdb_report` 应用路径 |
| 本地时间差异 | 日期目录来自 UDF 运行节点当前日期 | 集群节点时间应统一 |

## 十一、最终建议

建议只改 `RahaZipWebPublisher.java` 一个文件，不改三函数入口、不改 `RahaDetectionUdfService`、不改返回字段。

改造后行为应为：

1. 三函数继续按现有逻辑生成 ZIP。
2. 发布器先尝试 HDFS 上传。
3. 发布器再尝试参考 schmatch 的远端 SSH/SCP Web 发布。
4. 远端发布成功时，返回 `http://fmdbmanager-master-0:10030/fmdb_report/yyyy/M/d/<zip>`。
5. 远端发布失败时，降级到当前本地发布逻辑。

这样可以最小化改动范围，同时满足“ZIP 包通过 SSH 到另一台 `fmdbManagerPod`，并提供 URL 下载”的目标。
